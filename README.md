 Student Finance TrackerAn accessible, framework-free single-page application (SPA) engineered with semantic HTML5, mobile-first CSS3, and modular ECMAScript (ES6+) architecture. This application provides students with an intuitive, high-performance interface to log expenditures, query metrics via regular expressions, track custom budget caps with real-time screen-reader updates, and manage data migration through validated JSON files.🌐 Live Production Link: Deployments: Launch Live Web Application🎬 System Demonstration: Review System Demonstration on Loom🔬 System Features & ArchitectureFramework-Free Engineering: Built strictly using native browser APIs (Vanilla HTML, CSS, and JS) to ensure zero dependencies, high performance, and compliance with academic standards.Universal Input Security: Validates all database entries live via custom regular expression patterns, catching duplicate phrasing, invalid structures, and spacing issues as you type.Accessible Data Layouts: Features structural column headers (scope="col"), multi-directional alphanumeric sorting, and a fully non-destructive text highlighting system.Dynamic Budget Telemetry: Tracks current expenditures against an adjustable limit, automatically elevating system alerts to assistive screen-reader channels if spending thresholds break.JSON Portability Pipeline: Features structure-checked data migration workflows. This region automatically upgrades its severity setting to assertive if total spending breaks user-defined limits, forcing the screen reader to interrupt the user immediately with a critical warning message.#status-message (aria-live="polite"): Announces background actions when users perform mutations (e.g., "Transaction item successfully removed" or "Edits saved").#import-feedback (aria-live="polite"): Announces formatting outcomes when parsing files through data import channels..error-message (aria-live="polite" / role="alert"): Flags structural verification failures right next to form inputs while fields are modified.🎨 Color Contrast Compliance (WCAG AA)The color palette options are designed to provide sharp visual clarity and meet the strict contrast guidelines set by the WCAG AA compliance standards.Element AreaForeground ColorBackground ColorContrast RatioCompliance StatusPrimary Controls (Buttons like Add, Save)#FFFFFF (White)#111E30 (Dark Slate Blue)14.2:1✅ Exceeds AA ($> 4.5:1$)Main Readable Text (Labels, values, headers)#1A1A1A (Near Black)#FFFFFF (White)16.4:1✅ Exceeds AA ($> 4.5:1$)Secondary Controls (Clear Form actions)#111E30 (Dark Navy Text)#D2D7DF (Light Grayish Blue)10.5:1✅ Exceeds AA ($> 4.5:1$)Input Outlines & Placeholders#555555 (Slate Gray)#FFFFFF (White)6.2:1✅ Passes AA ($> 4.5:1$)System Alerts (Inline error messaging text)#D32F2F (Standard Red)#FFFFFF (White)4.7:1✅ Passes AA ($> 4.5:1$)💾 LocalStorage Serialization FormatApplication state structures are saved automatically to browser client memory arrays under the key namespace string sft:data.
